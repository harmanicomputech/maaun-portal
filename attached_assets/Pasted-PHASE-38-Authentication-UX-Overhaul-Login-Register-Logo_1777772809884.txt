PHASE 38: Authentication UX Overhaul (Login, Register, Logout, Password Reset)

OBJECTIVE:
Redesign and improve ONLY the authentication experience (login, register, logout, password reset) to be more modern, intuitive, and visually polished while preserving all existing backend logic.

DO NOT modify core authentication logic or role system.
ONLY improve UI/UX, flow, and user experience.

RIGHT PANEL (FORM):

4. LOGOUT CONFIRMATION FLOW
--------------------------------------------------

"Are you sure you want to logout?"

- clear session/token
- redirect to /login
- add fade-out transition

5. RESET PASSWORD FEATURE
--------------------------------------------------

Add "Forgot Password?" link on login page.

- input email

  "If an account exists, reset instructions have been sent"

- frontend-only mock flow is acceptable (no email service required yet)

--------------------------------------------------

Keep animations smooth and lightweight.

7. ERROR + SUCCESS UX IMPROVEMENT
--------------------------------------------------
